Five-star open data rating

Tim Berners-Lee's five-star framework measures how machine-accessible and interoperable open data is. This publication demonstrates what moving from three stars to four looks like in practice.

★★★★★ Available, structured, non-proprietary ✓ Achieved
Publicly accessible JSON-LD and HTML. No login required. No proprietary format.
★★★★★ Use open standards with dereferenceable URIs ✓ Achieved
Every record has a stable URI. Occupation links to O*NET. Content negotiation returns JSON-LD or HTML.
★★★★★ Link your data to other datasets Not yet
Requires a SPARQL endpoint and inbound links from external datasets. This is the next step.

Accessing the data

Browse a record

Each apprentice record has a human-readable HTML page and a machine-readable JSON-LD representation at the same URI.

# HTML — open in a browser http://localhost:7071/api/id/apprentice/1294199 # JSON-LD — request with Accept header curl -H "Accept: application/ld+json" \ http://localhost:7071/api/id/apprentice/1294199

List all records

# Paginated list — HTML http://localhost:7071/api/id/apprentice # Paginated list — JSON-LD curl -H "Accept: application/ld+json" \ "http://localhost:7071/api/id/apprentice?page=1&size=50" # Filter by SOC code curl -H "Accept: application/ld+json" \ "http://localhost:7071/api/id/apprentice?soc=47-2111.00"

What this demonstrates — and what's still missing

  • ⚠️
    RAPIDS has no SOC codes 1,328 free-text occupation strings had to be fuzzy-matched against the O*NET database. If RAPIDS published SOC codes natively, this pipeline would not exist.
  • ⚠️
    33.7% of RAPIDS records have no occupation Nearly 3 million of 8.75 million records could not be used because the occupation field was "Not Provided". A machine-readable SHACL data contract would catch this at ingest.
  • ⚠️
    No SPARQL endpoint This publication supports content negotiation and individual record dereferencing, but not federated SPARQL queries. That is the five-star step.

Source and provenance

This dataset is derived from the Apprenticeship Participant Characteristics dataset published by the U.S. Department of Labor, Employment and Training Administration on data.dol.gov.

Records were filtered to a 10,000-record sample using occupation strings with ≥95% SOC code match confidence against the O*NET database (version 30.2). Sampling was stratified by occupation to ensure broad coverage.
